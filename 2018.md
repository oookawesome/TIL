## 2018년 회고

### 1월~5월 MSA
2017년 하반기부터 해왔던 MSA 프로젝트가 18년 5월까지 이어졌다. 테스트 엔지니어 역할로 디바이스 서비스팀에 소속되어 업무를 수행했다. 애자일 프랙티스를 빡세게 적용하는 프로젝트로, TDD, 페어 프로그래밍 등의 프랙티스를 수행했다. 서비스팀의 스크럼과 회고, 개발 페어에 모두 참여했고, 단위테스트 코드도 페어로 같이 작성했다. 일주일 중 2일 정도를 함께 투입된 팀원들과 모여서 따로 업무를 수행했다. 프로젝트 전체를 대상으로 한 자동화 테스트(API Integration Test, E2E Test)를 개발, 관리하는 시간을 가졌다.

#### 좋았던 점
- (이전부터 간절히 원했던) 개발 프로젝트에 참여한 것 자체
- __개발 페어로 실제 스프링부트 REST 서버 개발을 해본 것__
- 서비스 개발팀의 분위기가 좋았음
- 애자일 개발 프랙티스를 경험한 것(IPM, Retro, Stand up, TDD, Pair programming)
- 함께 부서에서 투입된 3명의 동료가 좋았던 것
    - 후배에게 경험을 쌓을 기회를 많이 주었음(중요한 일도 시켜주었음..)
    - 테스트 자동화에 대한 역량이 뛰어나서 배울게 많았음
    - 개발 경험을 모두 가지고 있어 배울게 많았음
    - 성격이 좋아서 분위기가 좋고 트러블이 없었음
    - 학습 의욕을 고취시켜주었음
- 모든 종류의 자동화 테스트(Unit, API, GUI, Contract, Image..)를 거의 다 해본 것

#### 아쉬웠던 점
- 테스트 자동화 전문가라고 투입됬지만, 프로젝트에 전문성을 보여주거나, 큰 도움을 줬다거나, 알려줬다거나 한 부분이 거의 없었음. 서비스가 개발자가 더 잘했음
- 개발에 대해 너무 몰라서 창피했음
- 당연하게도 업무 평가가 높지 않았음
- 동료들에게 자신감이 부족했던 느낌을 주었던 것 같음
    - 모르는 게 너무 많아서 그랬나..?
- 프로젝트를 이해하고, 업무를 파악하고 1인분하기까지 너무 오래걸렸음
- 프론트엔드(리액트)를 거의 해보지 못했음

### 6월~12월 Robot Framework IDE 개발
대략 6월부터 연초부터 Robot Framework IDE를 개발해온 팀에 중간에 참여했다. Angular + Electron + Python 조합으로 구성된 프로젝트였다. 초반엔 소위 '바닥부터 쓸자' 정신으로 팀에서 하지 못하고 있던 주변 영역부터 참여했다. 젠킨스로 CI를 구성했고, 테스트 코드도 추가했다. Angular쪽 자바스크립트 단위테스트를 추가하고 싶었으나, Electron으로 동작되는 특성상 구현이 어려웠고, 대신 E2E 테스트 코드를 추가했다. 이 후 개발에 본격적으로 참여해, 신규 피처들에 대해 개발을 진행했다. 대부분 Angular로 돌아가는 프론트엔드 부분이었다. 클라이언트 프로그램이라는 속성과 Robot Framework기반이라는 속성때문에 인스톨러가 필요했는데, NSIS를 이용하여 인스톨러를 개발하는 작업도 진행했다.

#### 좋았던 점
- __Angular를 이용하여 프론트엔드 개발을 수행한 것__
- 새로운 기술(Electron, RxJS, NodeJS)을 경험해본 것
- JenkinsFile을 이용해 젠킨스 파이프라인을 주도적으로 구성한 것
- 타입스크립트 + Mocha 테스트코드를 작성해본 것 
- IDE 사용자들의 반응이 괜찮았던 것
- 인스톨러(NSIS)를 주도적으로 개발해본 것
- 자바스크립트, 웹, 브라우저 등의 웹개발 기본적 지식을 공부하고 익힌 것
- 팀 동료들과 분위기가 좋았던 것

#### 아쉬웠던 점
- 테스트 코드 작성한다고 시간을 많이 잡아먹어 본격 개발시작이 너무 늦었음
- 프로젝트 중간에 투입된 거라 초반의 설계와 준비과정에 참여하지 못함
- 백엔드(Python)는 이미 완성된 상태라 거의 손대지 못함
- 배워온 애자일 프랙티스를 많이 적용해보고 싶었는데 그렇게 하지 못했음
- 프론트엔드 및 주요 기술 스택관련 지식이 너무 없어 의견제시를 제대로 못했음
- 주요 개발 마무리된 후반부에 의욕이 떨어져 시간을 무의미하게 사용함
- 업무 평가가 높지 않았음....
